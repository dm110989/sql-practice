## 2. Переменные

### Что такое переменная

Переменная — это имя, под которым хранится значение.

Пример:

```python
app_name = "BiteNow"
users_count = 13450
average_rating = 4.7
is_operational = True
```

Здесь:

- `app_name` — строка (`str`)
- `users_count` — целое число (`int`)
- `average_rating` — дробное число (`float`)
- `is_operational` — логическое значение (`bool`)

### Зачем нужны переменные

Они нужны, чтобы:

- сохранить данные;
- использовать их позже;
- выполнять вычисления;
- не писать одно и то же много раз.

### Вывод на экран

```python
print(app_name)
print(users_count)
print("Рейтинг:", average_rating)
```

### Арифметика с переменными

```python
orders_today = 3200
yesterday_orders = 2870
average_order_price = 345.5

order_growth = orders_today - yesterday_orders
revenue_today = orders_today * average_order_price
```

### Полезные операторы

```python
10 + 5   # сложение
10 - 5   # вычитание
10 * 5   # умножение
10 / 5   # обычное деление
10 // 3  # целая часть от деления
10 % 3   # остаток от деления
10 ** 2  # степень
```

### Комментарий

- `/` даёт обычное деление;
- `//` отбрасывает дробную часть;
- `%` часто используют, когда надо проверить остаток.

Пример:

```python
boxes = 3200 // 500
remainder = 3200 % 500
```

### Быстрое изменение переменной

```python
users_count = users_count + 100
```

То же самое короче:

```python
users_count += 100
```

### Что запомнить

Переменная — это просто способ сохранить значение под именем.

## 3. Списки (`list`)

### Что такое список

Список хранит **несколько значений в одной переменной**.

Пример:

```python
orders_per_day = [2800, 3100, 2900, 3200, 3000]
cities = ["Москва", "Санкт-Петербург", "Казань"]
```

### Когда использовать список

Когда данных много и они относятся к одной группе:

- заказы по дням;
- оценки пользователей;
- города;
- id пользователей.

### Полезные функции для списков

```python
numbers = [5, 2, 8, 1]

len(numbers)      # количество элементов
sum(numbers)      # сумма
min(numbers)      # минимум
max(numbers)      # максимум
sorted(numbers)   # новый отсортированный список
list(reversed(numbers))  # список в обратном порядке
```

### Важный комментарий

`sorted(numbers)` **не меняет оригинальный список**, а создаёт новый.

Пример:

```python
numbers = [5, 2, 8, 1]
sorted_numbers = sorted(numbers)

print(numbers)         # [5, 2, 8, 1]
print(sorted_numbers)  # [1, 2, 5, 8]
```

## 4. Индексация в списках

### Как работает индексация

У каждого элемента списка есть номер — **индекс**.

В Python индексация начинается с **0**.

Пример:

```python
cities = ["Москва", "СПб", "Казань", "Тюмень"]
```

Тогда:

- `cities[0]` → `"Москва"`
- `cities[1]` → `"СПб"`
- `cities[2]` → `"Казань"`

### Отрицательные индексы

Можно обращаться с конца:

- `cities[-1]` — последний элемент
- `cities[-2]` — предпоследний

Пример:

```python
cities[-1]  # "Тюмень"
```

### Срезы

Срез — это взять часть списка.

```python
numbers = [10, 20, 30, 40, 50]

numbers[1:4]   # [20, 30, 40]
numbers[:3]    # [10, 20, 30]
numbers[2:]    # [30, 40, 50]
numbers[-3:]   # [30, 40, 50]
```

### Важное правило

Правая граница в срезе **не включается**.

То есть:

```python
numbers[1:4]
```

возьмёт элементы с индексами `1, 2, 3`.

### Частая ошибка новичков

```python
cities[4]
```

даст ошибку, если элемента с таким индексом нет.

Ошибка называется:

```python
IndexError
```

### Что запомнить

- первый элемент всегда `[0]`
- последний элемент удобно брать через `[-1]`
- в срезах правая граница не входит

## 5. `any()` и `all()`

Эти функции часто встречаются при работе со списками логических значений.

Пример:

```python
order_statuses = [True, True, False, True]
```

### `any()`

Возвращает `True`, если **хотя бы один** элемент истинный.

```python
any(order_statuses)  # True
```

### `all()`

Возвращает `True`, только если **все** элементы истинные.

```python
all(order_statuses)  # False
```

### Как запомнить

- `any` = хотя бы один
- `all` = все

## 6. Методы списков

Методы — это команды, которые вызываются **у самого списка**.

Общий вид:

```python
список.метод()
```

### `append()` — добавить в конец

```python
ratings = [5, 4, 4]
ratings.append(5)
print(ratings)
# [5, 4, 4, 5]
```

### `count()` — посчитать, сколько раз встречается значение

```python
ratings = [5, 4, 4, 5, 3]
ratings.count(5)
# 2
```

### `index()` — найти индекс первого вхождения

```python
ratings = [5, 4, 4, 5, 3]
ratings.index(3)
# 4
```

### Комментарий

`index()` находит **только первое совпадение**.

### `insert()` — вставить по индексу

```python
ratings = [5, 4, 5]
ratings.insert(1, 10)
print(ratings)
# [5, 10, 4, 5]
```

### `remove()` — удалить по значению

```python
ratings = [5, 4, 4, 5]
ratings.remove(4)
print(ratings)
# [5, 4, 5]
```

### Комментарий

`remove()` удаляет только **первое** найденное значение.

### `pop()` — удалить по индексу и вернуть значение

```python
ratings = [5, 4, 3]
value = ratings.pop(1)

print(value)    # 4
print(ratings)  # [5, 3]
```

### Почему `pop()` полезен

Он не просто удаляет, а ещё и возвращает удалённый элемент.\
Это удобно, если надо сохранить то, что убрали.

### `sort()` — сортирует сам список

```python
numbers = [5, 2, 8, 1]
numbers.sort()
print(numbers)
# [1, 2, 5, 8]
```

### Важная разница: `sort()` и `sorted()`

```python
numbers = [5, 2, 8, 1]

numbers.sort()        # меняет исходный список
sorted(numbers)       # создаёт новый отсортированный список
```

### `reverse()` — переворачивает сам список

```python
numbers = [1, 2, 3, 4]
numbers.reverse()
print(numbers)
# [4, 3, 2, 1]
```

## 7. Множества (`set`)

### Что такое множество

Множество — это набор **уникальных значений**.

Пример:

```python
user_ids = [101, 102, 103, 101, 104, 102]
unique_users = set(user_ids)
print(unique_users)
```

Результат будет без повторов.

### Когда множество полезно

Когда нужно:

- убрать дубликаты;
- быстро проверить наличие элемента;
- сравнить группы данных.

### Как создать множество

```python
colors = {"red", "blue", "green"}
```

или:

```python
numbers = set([1, 2, 2, 3, 3, 3])
# {1, 2, 3}
```

### Важный комментарий

Во множестве:

- нет повторов;
- нет порядка, как в списке;
- нельзя обратиться к элементу по индексу.

То есть так нельзя:

```python
numbers[0]
```

## 8. Методы множеств

### `add()` — добавить элемент

```python
delivery_cities = {"Москва", "Тюмень", "Казань"}
delivery_cities.add("Омск")
```

### `remove()` — удалить элемент

```python
delivery_cities.remove("Казань")
```

### `intersection()` — общие элементы

```python
users_yesterday = {101, 102, 103, 104}
users_today = {103, 104, 105}

common_users = users_yesterday.intersection(users_today)
print(common_users)
# {103, 104}
```

### `difference()` — что есть в одном, но нет в другом

```python
lost_users = users_yesterday.difference(users_today)
print(lost_users)
# {101, 102}
```

### `union()` — объединение уникальных значений

```python
all_users = users_yesterday.union(users_today)
print(all_users)
# {101, 102, 103, 104, 105}
```

## 9. Быстрое сравнение `list` и `set`

| Что сравниваем                 | `list`   | `set` |
| ------------------------------ | -------- | ----- |
| Хранит повторы                 | Да       | Нет   |
| Есть порядок                   | Да       | Нет   |
| Есть индексы                   | Да       | Нет   |
| Удобен для групп данных        | Да       | Да    |
| Удобен для уникальных значений | Не очень | Да    |

## 10. Частые ошибки новичка

### Ошибка 1. Путать `sorted()` и `sort()`

```python
numbers = [3, 1, 2]
sorted(numbers)
print(numbers)
```

`numbers` не изменится, потому что `sorted()` создаёт новый список.

### Ошибка 2. Ждать, что `remove()` удалит все одинаковые элементы

```python
numbers = [1, 2, 2, 2, 3]
numbers.remove(2)
print(numbers)
# [1, 2, 2, 3]
```

Удалился только первый `2`.

### Ошибка 3. Пытаться индексировать множество

```python
values = {1, 2, 3}
values[0]
```

Так нельзя, потому что у множества нет индексов.

### Ошибка 4. Забывать, что индексация идёт с нуля

```python
cities = ["Москва", "СПб", "Казань"]
print(cities[0])
# Москва
```
